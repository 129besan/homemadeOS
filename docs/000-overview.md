# homemadeOS 概要

## 目標

UEFI から起動し、Rust `no_std` カーネル、UEFI ブートローダー、
QEMU スモークテストを備える x86_64 学習用 OS を作る。

現在のベースラインは、QEMU 上で UEFI ブートローダーからカーネル ELF をロードし、
`kernel started` のシリアルログまで到達する段階である。ユーザーモード、syscall、
ファイルシステム、シェルは設計と一部コードがあるが、現在の確認済み baseline には
含めていない。

## アーキテクチャ

```
アーキテクチャ: x86_64
起動方式: UEFI
カーネル: モノリシック
言語: Rust + アセンブリ
メモリモデル: ハイヤーハーフカーネル (0xffff_8000_...)
プロセスモデル: ユーザー空間とカーネル空間を分離する予定
実行形式: ELF64
ファイルシステム: initramfs -> 単純な FS
スケジューラ: プリエンプティブ・ラウンドロビンを目標にする
対象エミュレータ: QEMU
```

## 対象外

- Linux 互換性
- SMP
- 動的リンク
- ジャーナリングファイルシステム
- 本番環境水準のセキュリティ

## マイルストーン

1. 起動: UEFI ブートローダーからカーネルへ制御を渡す
2. カーネルログ: シリアル出力を動かす
3. BootInfo: memory map と framebuffer 情報を渡す
4. 物理メモリ: フレームアロケータを実装する
5. ページング: ページテーブルを管理する
6. 割り込み: IDT、例外、タイマーを実装する
7. スケジューリング: カーネルスレッドとプリエンプションを実装する
8. ユーザーモード: ring 3 へ移行する
9. システムコール: ユーザー空間からカーネルへのインターフェースを実装する
10. Initramfs: 組み込みファイルシステムを実装する
11. ELF ローダー: ユーザープログラムを起動する
12. シェル: 対話的なコマンドラインを実装する
